\begin{abstract}
Este relatório apresenta a implementação e análise comparativa entre o algoritmo de Dijkstra e uma heurística gulosa para o problema de caminho mínimo em grafos direcionados ponderados. Os experimentos foram realizados em grafos completos de tamanhos variados e em instâncias do Moodle com 10.000 e 1.000.000 de vértices. O objetivo é comparar o número de operações de relaxamento (comparações) realizadas por cada algoritmo. Os resultados demonstram que, embora a heurística gulosa realize menos comparações em grafos esparsos, ela alcança apenas 0,28\% dos vértices na instância de 10k e 0,012\% na instância de 1M, confirmando que não é adequada para o problema de caminho mínimo de origem única (SSSP).
\end{abstract}

\section{Introdução}

O problema de caminho mínimo de origem única (SSSP -- \textit{Single Source Shortest Path}) consiste em encontrar as distâncias mínimas de um vértice origem para todos os demais vértices de um grafo ponderado. Este trabalho implementa e compara:

\begin{itemize}
    \item \textbf{Algoritmo de Dijkstra}: algoritmo clássico e ótimo para grafos com pesos não-negativos, com complexidade $O((V + E) \log V)$ usando heap binário.
    \item \textbf{Heurística Gulosa}: estratégia que, em cada passo, escolhe a aresta de menor peso saindo do conjunto de vértices já alcançados. Esta heurística \textbf{não garante} caminho mínimo ótimo em geral.
\end{itemize}

A métrica de comparação utilizada é o \textbf{número de comparações de relaxamento}, contabilizadas pela operação \texttt{dist[u] + w < dist[v]}.

\section{Metodologia}

\subsection{Estrutura de Dados}

Os grafos foram representados utilizando \textbf{listas de adjacência}, armazenadas em um dicionário Python:
\begin{lstlisting}
Adj = Dict[int, List[Tuple[int, float]]]
adj[u] = [(v1, w1), (v2, w2), ...]
\end{lstlisting}

Onde \texttt{u} é o vértice de origem, \texttt{v} é o vértice de destino e \texttt{w} é o peso da aresta $(u,v)$.

\subsection{Implementações}

\subsubsection{Dijkstra}

\begin{lstlisting}
def dijkstra_count(adj: Adj, s: int) -> DijkstraResult:
    dist = {u: inf for u in adj}
    parent = {u: None for u in adj}
    dist[s] = 0.0
    
    heap = [(0.0, s)]
    visited = set()
    comparisons = 0
    
    while heap:
        du, u = heapq.heappop(heap)
        if u in visited:
            continue
        visited.add(u)
        
        for v, w in adj[u]:
            comparisons += 1  # Contagem
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                heapq.heappush(heap, (dist[v], v))
    
    return DijkstraResult(dist, parent, comparisons)
\end{lstlisting}

\subsubsection{Heurística Gulosa}

\begin{lstlisting}
def greedy_sssp_edge_first(adj: Adj, s: int) -> GreedyResult:
    dist = {u: inf for u in adj}
    parent = {u: None for u in adj}
    dist[s] = 0.0
    
    visited = set([s])
    edge_heap = []  # (peso, u, v)
    
    for v, w in adj[s]:
        heapq.heappush(edge_heap, (w, s, v))
    
    comparisons = 0
    
    while edge_heap:
        w, u, v = heapq.heappop(edge_heap)
        
        comparisons += 1  # Contagem
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            parent[v] = u
            
            if v not in visited:
                visited.add(v)
                for x, wx in adj[v]:
                    heapq.heappush(edge_heap, (wx, v, x))
    
    return GreedyResult(dist, parent, comparisons)
\end{lstlisting}

\subsection{Testes Realizados}

\subsubsection{Grafos Completos (Itens 2.II.a-e e 3.II.a-e)}

Foram gerados grafos completos direcionados com $n = 4$ até $n = 1000$ vértices. Cada aresta possui peso positivo aleatório gerado com semente fixa (\texttt{seed=42}) para reprodutibilidade. Para cada grafo:
\begin{enumerate}
    \item Aplicamos Dijkstra partindo do vértice 0.
    \item Aplicamos a heurística gulosa partindo do vértice 0.
    \item Contamos o número de comparações em cada caso.
    \item Plotamos um gráfico comparativo.
\end{enumerate}

\textbf{Observação}: grafos completos possuem $O(n^2)$ arestas. Os testes foram realizados até $n=1000$, gerando aproximadamente 999.000 comparações.

\section{Análise e Discussão}

\subsection{Complexidade Teórica vs. Experimental}

\begin{itemize}
    \item \textbf{Dijkstra}: $O((V + E) \log V)$ com heap binário. Em grafos completos, $E = O(V^2)$, resultando em $O(V^2 \log V)$.
    \item \textbf{Heurística Gulosa}: Também utiliza heap, mas a estratégia de escolha de arestas leva a comportamentos distintos dependendo da topologia do grafo.
\end{itemize}

\subsection{Grafos Completos: Comportamento Idêntico}

Nos grafos completos testados ($n=4$ até $n=1000$), ambos os algoritmos apresentaram \textbf{exatamente o mesmo número de comparações}:
\begin{itemize}
    \item A fórmula observada é $C(n) = n(n-1)$, correspondendo ao número total de arestas no grafo completo direcionado.
    \item Este comportamento ocorre porque, em grafos completos com pesos positivos, ambos os algoritmos acabam explorando todas as arestas.
    \item A diferença fundamental está na \textbf{corretude}: Dijkstra garante caminho mínimo, enquanto a gulosa não garante.
\end{itemize}

\subsection{Instâncias Esparsas: Diferença Drástica}

Nas instâncias do Moodle (grafos esparsos), observamos comportamento radicalmente diferente:

\begin{enumerate}
    \item \textbf{Dijkstra}: comportamento robusto e completo
    \begin{itemize}
        \item Alcança todos os vértices acessíveis a partir da origem
        \item 10k vértices: 123.462 comparações, 10.000 vértices alcançados
        \item 1M vértices: 15.172.126 comparações, 1.000.000 vértices alcançados
        \item Garante caminho mínimo para todos os vértices
    \end{itemize}
    
    \item \textbf{Heurística Gulosa}: falha crítica
    \begin{itemize}
        \item Alcança apenas uma fração mínima dos vértices
        \item 10k vértices: 424 comparações, apenas 28 vértices (0,28\%)
        \item 1M vértices: 1.847 comparações, apenas 120 vértices (0,012\%)
        \item A estratégia gulosa de escolher sempre a menor aresta disponível cria caminhos localmente ótimos que rapidamente ficam "presos" em regiões do grafo
        \item Não garante corretude nem completude
    \end{itemize}
\end{enumerate}

\subsection{Corretude e Adequação}

\begin{itemize}
    \item \textbf{Dijkstra}: garante caminho mínimo correto para grafos com pesos não-negativos. É o algoritmo adequado para o problema SSSP.
    \item \textbf{Gulosa}: \textbf{não garante} corretude em geral. Os experimentos mostram que:
    \begin{itemize}
        \item Em grafos completos: encontra caminhos (não necessariamente mínimos) para todos os vértices
        \item Em grafos esparsos: falha drasticamente, alcançando menos de 1\% dos vértices
        \item Não é adequada para resolver o problema SSSP
    \end{itemize}
\end{itemize}

\subsection{Implicações Práticas}

Os resultados demonstram que:
\begin{enumerate}
    \item A métrica de "número de comparações" isolada não é suficiente para avaliar qualidade de algoritmos para SSSP
    \item É essencial verificar \textbf{corretude} (caminhos mínimos corretos) e \textbf{completude} (alcançar todos os vértices acessíveis)
    \item A heurística gulosa, apesar de simples e rápida em grafos esparsos, não é confiável para o problema geral
    \item Dijkstra permanece como solução robusta e correta para SSSP com pesos não-negativos
\end{enumerate}

\section{Conclusão}

Este trabalho implementou e comparou o algoritmo de Dijkstra com uma heurística gulosa para o problema de caminho mínimo. Os resultados revelam diferenças fundamentais entre as abordagens:

\begin{enumerate}
    \item \textbf{Grafos Completos}: Ambos os algoritmos realizaram o mesmo número de comparações ($n(n-1)$), mas apenas Dijkstra garante corretude dos caminhos.
    
    \item \textbf{Grafos Esparsos}: A heurística gulosa apresentou falha crítica, alcançando menos de 1\% dos vértices nas instâncias de teste (28/10.000 e 120/1.000.000), enquanto Dijkstra alcançou todos os vértices acessíveis.
    
    \item \textbf{Número de Comparações}: A métrica isolada de comparações não é suficiente para avaliar qualidade. É essencial verificar corretude e completude da solução.
    
    \item \textbf{Adequação}: Dijkstra é o algoritmo adequado para SSSP com pesos não-negativos, apresentando complexidade $O((V+E)\log V)$, corretude garantida e exploração completa do grafo.
    
    \item \textbf{Lição Fundamental}: Estratégias gulosas simples podem falhar drasticamente em problemas que exigem otimização global. A escolha local da menor aresta não garante caminho mínimo global.
\end{enumerate}

Os experimentos confirmam os resultados teóricos da literatura: o algoritmo de Dijkstra é a escolha correta para o problema de caminho mínimo com pesos não-negativos, oferecendo garantias de corretude que heurísticas simples não podem fornecer.
